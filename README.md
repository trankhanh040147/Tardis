# Tardis

Tardis is an interactive learning workflow for working engineers who want to understand new concepts through practice instead of passive consumption. It uses short, focused AI sessions to surface misconceptions quickly, correct weak mental models in real time, and help knowledge stick through repeated review.

As engineers learn, Tardis turns each session into a structured record of progress: concepts are stored as ordered markdown notes, grouped by category, and connected to a roadmap that reflects real skill growth over time. The goal is to make engineering learning active, honest, and durable so engineers can adapt faster, build confidence grounded in actual understanding, and explore adjacent domains without getting trapped in tutorial hell.

## Mission

Help working engineers expose misconceptions fast and make new engineering concepts stick.

## Goal

Create the fastest feedback loop for active engineering learning and long-term retention.

## Core Principles

- Action over theory
- Retention over consumption
- Short, focused practice sessions
- Direct feedback over vague encouragement
- Adaptability over static knowledge
- Exploration across adjacent engineering domains

## How Tardis Works

- Store learned concepts as ordered markdown files for later review
- Organize knowledge by category and topic
- Track skill growth through a roadmap tree
- Run interactive Q&A sessions with immediate feedback
- Identify guessing, weak intuition, and misconceptions early
- Suggest areas to practice based on gaps and progress

## Philosophy

### Action Over Theory

Tardis is built around practice. The goal is not to consume more content, but to test understanding, reveal weak mental models, and strengthen them through interaction.

### Retention Over Consumption

Learning only matters if it lasts. Tardis stores what was learned so concepts can be revisited, reviewed, and reinforced over time.

### Tech Rot Is Real

Engineering knowledge changes constantly. Tardis focuses on helping engineers adapt quickly rather than optimizing for static memorization.

### Kill Tutorial Hell

Passive reading often creates false confidence. Tardis pushes learning into active recall, correction, and repetition.

### Time Efficiency

Sessions should fit into a coffee break. The workflow is designed for minimal friction: get in, practice, get feedback, get out.

### Radical Candor

Feedback should be clear and objective. Tardis corrects misconceptions immediately instead of hiding uncertainty behind polite but unhelpful responses.

### Breaking Silos

Engineers often avoid unfamiliar adjacent domains because the entry cost feels high. Tardis aims to make those domains easier to approach and less intimidating to practice.

## Intended Experience

- Fast sessions with clear learning intent
- Honest correction when the user is guessing
- Flexible topic selection or AI-guided practice
- Persistent notes that build a personal knowledge base
- A visible roadmap of growth across categories

## In One Line

Tardis helps working engineers learn engineering concepts the hard way on purpose, so the knowledge actually stays.
